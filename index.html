Welcome

This is a collection of Product Management processes/tools I've had success with and a few recent deliverables.

Tools/Processes - the Planning Framework

The Templates contain a model that I've found helpful for generating useful information for a feature, specifically defining and publishing the 'how', 'why' and 'what' context relevent to the customer(s) and development team alike.

The templates, and the process of filling them out, make-up the Planning Framework. It enables a Product Manager to proactively meet the needs of all stakeholders (Execs, Customers, Dev Team) as it's a more holistic approach to defining a feature, or collection of features (project), at the Epic-level.

Deliverables:

Monitoring- As an application team, I need an automated solution that enables monitoring of cloud-based resources so that I'm enabled to create custom dashboards, reports and alerts and respond in a way that reflects the needs of my customer.

Secrets Management - As an engineer developing an application for the CI / CD Pipeline, I want a secret management solution to store secrets so that I have an easy way to manage secrets across account demarcation boundaries. As an Operations stakeholder I want the ability to change account passwords and keys without having to engage development or devops personal or executing the CI/CD Pipeline so that we can maintain a strong security presence and best practices.

Blue-Green Deployment - As a application developer, I want Blue/Green Deployment capabilities so that I can manage application deployments with greater risk mitigation's for potential application issues.

AMI Builds - As an application team, I need a centralized pipeline that builds secure AMIs so that I can focus on building my application.
